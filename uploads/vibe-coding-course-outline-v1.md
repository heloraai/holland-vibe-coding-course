# Vibe Coding 零基础课程 · 教学大纲 v1.0

> 给完全没写过代码、但有点小想法的人。
> 学完之后你会拥有一个上线的小项目，以及继续自己走下去的能力。

---

## 为什么做这门课

过去做一个 App 要学三年代码；现在，一个小白从"有想法"到"网上有人在用"可以压缩到一周。改变发生了，但很多人还不知道路怎么走。

这门课不是教你"用某一个工具"，是带你走一次完整的**从 concept 到上线**的链路。工具会变，链路不变。

## 适合谁

- 没写过代码、没碰过命令行
- 脑子里经常冒出"要是有个 X 就好了"的想法
- 能搞定电脑基本操作（装软件、注册账号）
- 想做出能发给别人看、真实有人在用的东西

## 学完你会拥有

1. 一个挂在公网、有 URL 的小项目
2. 一条能自己再走一遍的完整 workflow
3. 一张"下一步学什么"的地图

## 课程三原则

1. **先讲明白，再做明白** — 前半程只看、不急着动手
2. **从概念到上线** — 不讲半截，走完整条链路
3. **品味比语法重要** — Vibe Coding 时代，这句话会在课里反复出现

---

## 课程地图

| 阶段 | 章节 | 主旨 | 预计时长 |
|---|---|---|---|
| **Part 1 · 心法与准备** | Ch 1–2 | 搞懂是什么、想清楚做什么 | ~2h |
| **Part 2 · 跟着 SleepForest 走一遍** | Ch 3–6 | 拆解一个完整项目，看懂每一步 | ~6h |
| **Part 3 · 自己做一遍** | Ch 7–8 | 从白纸到上线 + 下一步路线图 | ~6h |

**SleepForest（Sleepy Sprites）是贯穿全课程的案例项目**——一个 4 个场景 + 4 只动物 + 4-7-8 呼吸法的助眠 App，用 React + Gemini API 做，部署在 Vercel。每章都会拆出其中一部分来讲。其他案例会在 Ch 1 和附录 B 补充，保证学员视野不被单一案例框住。

# Part 1 · 心法与准备

## 第 1 章 · Vibe Coding 是什么

**⏱ 约 45 分钟** · **难度：☆☆☆☆☆** · **动手：无**

### 本章目标
听完之后你会有一种"这事儿我能干"的感觉 — 不是被术语吓跑，而是跃跃欲试。

### 核心内容

1. **Vibe Coding 这个词的来头**
   Karpathy 那条推的原文、背景、为什么一下子火了。
2. **从"写代码"到"描述意图"**
   一个对比：传统开发怎么做一个按钮 vs Vibe Coding 怎么做。
3. **为什么是现在**
   模型能力、工具链、成本，三条曲线相遇。
4. **案例欣赏（不是要你学会，是要你有感觉）：**
   - SleepForest · 一个助眠 App（本课程主例）
   - "今晚吃什么"轮盘 · 纯静态小工具
   - AI 读书笔记整理器 · 接 AI 的个人工具
   - "生日倒数 + 心愿记录"个人页 · 单页互动
5. **这门课走完你手上会有什么**

### 本章交付物
无作业。进入 Ch 2。

---

## 第 2 章 · 找到你的 Concept

**⏱ 约 1 小时** · **难度：★☆☆☆☆** · **动手：写一页纸**

### 本章目标
离开本章时，你会有一个用一句话讲得清楚的项目想法。

### 核心内容

1. **Idea 从哪里来 — 三条挖掘路径**
   - 自己的痛点（什么事情每天都烦你一下）
   - 自己的兴趣（什么事情周末你愿意搞 3 小时）
   - 身边人的抱怨（哪怕只是一句"哎这玩意怎么这么难用"）
2. **MVP 思维 — 压到一句话：**
   「一个 ___ 的 ___，用来 ___」
3. **第一个项目的"体检标准"：**
   - ✅ 一个人自己能用
   - ✅ 不需要登录账号
   - ✅ 不需要记住历史（刷新清零可接受）
   - ⚠️ 这些限制后续章节会慢慢放宽
4. **SleepForest 反推演示：**
   - 原始感受："有时候晚上睡不着，想要有人陪着呼吸"
   - 压缩 pitch："一个会陪你做 4-7-8 呼吸法、讲睡前故事的小动物助眠 App"
   - 核心 3 功能：场景选择 / 呼吸引导 / 角色独白
   - **砍掉的功能：** 用户登录、历史记录、付费订阅、多语言 — 每一刀都讲为什么砍
5. **3 个常见死法：**
   - 想做太大（"比微信还好用"）
   - 想做太抽象（"让人开心的东西"）
   - 被已有产品吓住（"X 已经做过了"）

### 章末交付物
**项目 1-Pager**：
- 一句话 pitch
- 最核心 3 个功能
- 砍掉的 3 个功能
- 3 个用户场景想象

# Part 2 · 跟着 SleepForest 走一遍

## 第 3 章 · 用 Google AI Studio 做原型

**⏱ 约 90 分钟** · **难度：★★☆☆☆** · **动手：出一个可点击原型**

### 本章目标
在浏览器里做出一个能点、能交互的原型。这一步**不写一行代码**。

### 核心内容

1. **为什么从原型开始**
   你的想法 vs 别人看到的东西，90% 的时候不一样。先把"不一样"暴露在屏幕上。
2. **AI Studio 入门三步**
   打开 → 选模型 → 第一次对话
3. **好 prompt vs 坏 prompt（逐句对比）**
   - 坏：「做一个助眠 App」
   - 好：「做一个深蓝色调的助眠 App 首页：中间有一只坐在雨夜窗边的卡通猫，底部中央是一个圆形播放按钮，下方有 4 个场景切换按钮（雨、森林、海、教室），整体氛围安静、偏暖蓝」
4. **给 AI 风格参考的三种方式**
   - 文字描述（"吉卜力"、"Monument Valley 的配色"）
   - 参考产品名
   - 直接贴截图
5. **迭代节奏：一次只改一件事**
6. **SleepForest 原型复盘**
   最初 prompt、5 次关键迭代、第 1 版到最终版的对比。

### 章末交付物
把 Ch 2 的 1-Pager 跑成一个可交互原型。录 1 分钟屏幕演示。

### 卡住了怎么办
- AI 听不懂你说的 → 用"参考产品 + 截图 + 具体元素"三件套
- 越改越乱 → 回上一个版本，别叠 bug
- 根本不知道要长什么样 → 找 2–3 个你喜欢的产品截图贴进去

## 第 4 章 · 认识 Codex

**⏱ 约 90 分钟** · **难度：★★☆☆☆** · **动手：第一次让 AI 改你的代码**

### 本章目标
第一次让 Codex 改动一个真实项目，理解"对话式编程"是什么体验。

### 核心内容

1. **Codex 是什么**
   一句话：在你的电脑终端里，能读你文件、改你文件、跑你代码的 AI。
   和 ChatGPT 的区别：ChatGPT 只会"说"，Codex 会"做"。
2. **环境准备（每一步截图）**
   - 装 Node.js
   - 打开终端（Mac / Windows 分开讲）
   - 装 Codex CLI
   - 在项目目录启动它
3. **第一次对话实操（用 SleepForest 当练习场）**
   - 让它读一遍项目，复述一下里面有什么
   - `npm install` + `npm run dev` 跑起来
   - 第一个改动：默认时长从 30 分钟改成 15 分钟
4. **心法：描述任务，不是写代码**
   - 不说："把 App.tsx 第 29 行的 30 改成 15"
   - 而说："默认时长从 30 分钟改成 15 分钟"
5. **Approve / Reject 怎么判断**
   - 什么时候可以放过（不看代码也 OK）
   - 什么时候必须逐字看（涉及删除、重构、接外部服务）

### 章末交付物
在 SleepForest 项目上做 3 个指定改动：换一句文案、换一个主色、加一个新按钮。截图提交。

### 卡住了怎么办
- Codex 改错了 → 让它"撤销刚才的改动"，换一种说法重试
- 改完跑不起来 → 把报错原文贴回去给它，它自己会修
- 它答非所问 → 说明你的描述太模糊；加"我希望的结果是 ___"这句上下文

## 第 5 章 · 从原型到完整产品（含调试与打磨）

**⏱ 约 3 小时** · **难度：★★★☆☆** · **动手：把原型补全到能用**

全课最长最重的一章。也是 SleepForest 会被拆得最彻底的一章。

### 本章目标
看懂一个完整前端项目怎么组成、怎么一步步把原型长大成产品、并学会调试的基本心法。

### 核心内容

#### A · SleepForest 解剖（项目导览）
- `App.tsx` — 大脑，管所有状态和流程
- `components/` — 可复用的"乐高积木"：`Character` / `SceneSelector` / `BreathingGuide` / `StressBar` / `SceneBackground`
- `services/geminiService.ts` — 和 AI 说话的那一小块
- `public/sounds/` — 音频资源
- `types.ts` — "标签"定义（标签让 AI 和你说同一种话）
- `package.json` — 项目的依赖清单

#### B · 三个最小必要概念（讲清楚就行，别深究）
- **组件** — 可复用的积木块
- **状态** — 当前在播 / 暂停中，这一类"会变"的东西
- **副作用** — 时间在流动、音频要播、数据要从外部拿 — 这类跟外部世界打交道的事

#### C · 接入第一个 AI 能力
拆 SleepForest 里 `geminiService.ts` 的全链路：prompt 怎么写 → API 怎么调 → 拿到的文本怎么塞进界面。
一句话：接 AI 其实就 20 行代码。

#### D · 喂任务给 Codex 的正确姿势 — 拆步走
反例："帮我做完整个 App"（= 灾难）

正例：
1. 先让界面在本地跑起来
2. 加场景切换
3. 套上呼吸引导
4. 接入 AI 生成文本
5. 加音频和动画

**每一步的"验收标准"：** 这一步做完了怎么判断？

#### E · 调试心法
- 报错了：原文贴回去，一个字都别改
- 体验怪：学会描述"感觉"（"切场景时音频要 1 秒才跟上，希望画面和声音同时变"）
- 小改 vs 大改：超过 3 次小改还不行，就让 Codex 开新分支整段重写

#### F · 品味是你的护城河
- Vibe Coding 时代，语法门槛几乎为零；审美反而是最稀缺的能力
- SleepForest 三个真实打磨案例：
  - 呼吸动画从"线性"改成"缓动"后立刻舒服了
  - 场景切换时音频短断 → 怎么发现、怎么描述、怎么修
  - 最后 60 秒音量渐弱这个细节从何而来

### 章末交付物
你的项目：
1. 能在本地跑起来
2. 核心交互跑通
3. 做过至少一次明确的调试修复
4. 至少 5 个"审美打磨"细节

### 卡住了怎么办
- 一步跑不通、后面都塞住了 → 立刻回退，别硬冲
- 越改越乱、不知道当前状态 → 提交一次代码作为安全点（让 Codex 帮你做）
- 想放弃了 → 这很正常；找朋友讲一遍你在做什么，往往会重新点燃

## 第 6 章 · 什么是 Vercel？部署上线

**⏱ 约 90 分钟** · **难度：★★☆☆☆** · **动手：拿到一个公开 URL**

### 本章目标
你的项目从"只在我电脑上能跑"变成"全世界都能打开的一个链接"。

### 核心内容

#### A · 先搞懂什么是"部署"
- 之前：项目只活在你电脑里
- 之后：一个 `https://...` 链接，发给任何人都能打开

#### B · Vercel 是什么
- 一句话：一个专门帮前端项目上线的平台
- 它替你做了什么：自动拉代码、自动构建、分发到全球 CDN、配好 HTTPS
- 为什么不自己买台服务器：贵、麻烦、还要运维；Vercel 免费额度对小项目够用很久
- 它和 Netlify / Cloudflare Pages 是同类产品（一句话带过）

#### C · Git 和 GitHub 最小必要知识
只讲 4 件事：建 repo、commit、push、看历史。
全程让 Codex 帮你跑命令，不逼你背 git。

#### D · Vercel 操作全流程（跟做截图）
1. 用 GitHub 注册 Vercel
2. Import 你的项目
3. 第一次部署（点一下按钮，等 2 分钟）
4. 拿到 `xxx.vercel.app` 链接

#### E · 环境变量（重点，关乎安全）
- 为什么 API Key 不能写在代码里（暴露 = 别人盗用 + 花你钱）
- SleepForest 的 Gemini API Key 怎么配
- 在 Vercel 后台设置环境变量的步骤

#### F · Preview Deployment — Vercel 的杀手锏
- 每次 push 都得到一个新链接
- 可以在不影响正式版的情况下把"实验版"发给朋友试
- 觉得 OK 就 merge，不 OK 就扔掉

#### G · 加餐：自定义域名（非必需）

### 章末交付物
一个真实的 `https://` 链接。发给 3 个真实朋友试用，收集至少 3 条反馈。

### 卡住了怎么办
- 部署报错 → 把 Vercel 构建日志贴给 Codex，它会修
- 部署成功但白屏 → 八成是环境变量没配；检查 Vercel Settings → Environment Variables
- 朋友说"打不开" → 先用手机流量（不连 Wi-Fi）打开一次试试

# Part 3 · 自己做一遍

## 第 7 章 · 动手实战 · 从零做你自己的项目

**⏱ 约 4–6 小时** · **难度：★★★★☆** · **动手：全部**

**这是你的大作业。** 前 6 章看懂的东西，这一章要自己独立走一遍。

### 本章目标
你交出：一个上线的、能分享的、你自己从零做的小项目。

### 工作流 · 8 个 Stage

**Stage 0 · Kickoff**
回看 Ch 2 的 1-Pager。问自己一次："我现在还想做这个吗？"
— 不想做了就改；改了就继续。

**Stage 1 · 重做一次原型（AI Studio）**
带着 Ch 3 学到的全部东西回到 AI Studio。

**Stage 2 · 拆功能清单**
把项目拆成 5–8 个小步骤，每一步的验收标准写清楚。

**Stage 3 · 用 Codex 一步步搭起来**
每完成一步本地跑一遍，别积压未验证的代码。

**Stage 4 · 打磨 Sprint**
至少 5 个审美 / 交互细节改进。

**Stage 5 · 部署上线**
推 GitHub → Vercel 自动部署 → 配环境变量 → 拿到公开链接。

**Stage 6 · 发给 3 个真实用户**
不要只发给家人（他们会说"好棒"）。找会给你真实反馈的人。

**Stage 7 · 一轮迭代**
根据反馈改至少 3 处，重新部署。

### 三条备选项目路线（给想不出 idea 的人）

| 路线 | 难度 | 特征 | 示例 |
|---|---|---|---|
| **A · 纯静态工具** | ★★☆☆☆ | 完全不接 API | 待办清单、习惯打卡、个人主页 |
| **B · 带 AI 能力的小工具** | ★★★☆☆ | 接 Gemini/OpenAI | AI 每日一言、AI 图书总结、AI 朋友圈文案助手 |
| **C · 单页游戏 / 互动** | ★★★☆☆ | 纯前端互动 | 小型 2048、打字练习、冥想呼吸可视化 |

### 章末交付物
1. 一个 `https://...` 的公开 URL
2. 一份 README（项目是干嘛的 + 灵感来自哪 + 开发花了多久）
3. 至少 3 位真实用户的反馈记录

### 卡住了怎么办（本章专属）
- **卡在 Stage 2 不知道怎么拆** → 回 Ch 5 看 SleepForest 的拆法，一一对照
- **Stage 3 写出来不是你想要的** → 回原型重看一遍，问自己"到底想要哪个细节不一样"
- **整个 Stage 做崩了想放弃** → 允许自己做一个更小的版本。完成比完美重要

## 第 8 章 · 继续 Vibe 下去

**⏱ 约 1 小时** · **难度：-** · **动手：规划 + 分享**

### 本章目标
你知道课程结束之后接下来 3 个月自己可以怎么走。

### 核心内容

#### A · 从"一次性作业"到"真产品"的正循环
反馈 → 迭代 → 再反馈。这个循环怎么启动、怎么保持节奏。

#### B · 能力地图（以 SleepForest 为原点）
朝不同方向各走一步，会遇到什么：
- 登录与用户系统
- 数据持久化（就是我们这期略过的 Supabase 这类工具）
- 付费订阅（Stripe）
- 移动端（PWA、原生）
- 多用户协作
- 更复杂的 AI 能力（语音、图像、视频、Agent）

#### C · 工具换代节奏
每 3–6 个月生态会翻一次。怎么保持跟进而不焦虑：关注 2–3 个信息源就够。

#### D · 社区和信息源推荐
（下一轮迭代时补具体清单）

#### E · 结课仪式
在课程社群里发自己的项目 URL + 一段 30 秒的作者故事。
